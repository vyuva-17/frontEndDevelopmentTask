Task1: (Setup Angular Routing for 3 Pages [1 Page without authentication, 2 Pages with Authentication using CanActivate])

Functionality

I have implemented this using CanActivate API. In this User Token is stored in local Storage as userCredentials. AuthGaurdServices will check wether the local Storage Key is set or not if it is not set then CanActivate API will route to the About Component otherwise you can navigate to all tabs. You can check this by removing the userCredentials Key in localStorage this key will set when the Application gets loaded.

Component

auth-gaurd.services.ts,app-module.ts,app-routing.ts this are files which contains major functionality for task1 implementation

Task2: (Use HttpInterceptor to set header token)

Functionality

I have implemented this using Http_Interceptor API. In this I have created interceptor to handle request and response headers for all requset and response passed/get from the Rest service. Using Http_Interceptor API it is easy to maintain all request and response header handling in one place. You check this in Task Implementation Page when page loads request will pass with Authorized token to get employee data.

Task4: (Pass Message between two Components)

Functionality

I have implemented this using @Input Decorator. In this I have get the employee data from rest services and passed the data stream to the child component(display-employee-child.component) from parent component(task-implementation.component) data stream received and passed through data binding and @Input directive to render the data in child component(display-employee-child.component) rendered data is displayed in parent component(task-implementation.component) .
